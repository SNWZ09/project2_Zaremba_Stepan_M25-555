# project2_Zaremba_Stepan_M25-555

Данный проект представляет собой создание и взаимодействие с примитивной базой данных.


## Управление таблицами

    Функции:
    create_table <имя_таблицы> <столбец1:тип> <столбец2:тип> .. - создать таблицу
    ist_tables - показать список всех таблиц
    drop_table <имя_таблицы> - удалить таблицу
    exit - выход из программы 
    help - справочная информация

Также в коде присутствуют следующие декораторы:
    1. С помощью декоратора `@handle_db_errors` вся логика обработки стандартных ошибок (отсутствие таблицы, неверные данные) вынесена из основного кода.
    2. Для предотвращения случайного удаления данных был создан декоратор `@confirm_action`. Он  запрашивает у пользователя подтверждение (`y/n`) перед выполнением таких команд, как `DROP TABLE` и `DELETE`.
    3. Декоратор `@log_time` используется для "медленных" операций (чтение файла, вставка), чтобы отслеживать их производительность.
    4. Кэширование запросов. Для ускорения повторных `SELECT`-запросов используется механизм кэширования, построенный на замыканиях.
